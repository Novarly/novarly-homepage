Our Commitment to Accessibility

Novarly is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and apply relevant accessibility standards to ensure our Services are accessible to all users.

Accessibility Standards

We strive to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards and follow accessibility best practices, including:

WCAG 2.1 Level AA Compliance

  • Perceivable: Information and UI components are presentable in ways users can perceive
  • Operable: UI components and navigation are operable by all users
  • Understandable: Information and UI operation are understandable
  • Robust: Content works with a variety of assistive technologies

Platform-Specific Guidelines

  • iOS: Adherence to Apple's Accessibility Guidelines
  • Android: Compliance with Android Accessibility Guidelines
  • Web: Following W3C Web Accessibility Initiative standards

Accessibility Features

Website Accessibility

Our website includes:

  • Keyboard Navigation: Full functionality accessible via keyboard
  • Screen Reader Support: Compatible with NVDA, JAWS, and VoiceOver
  • High Contrast Mode: Enhanced visual contrast for better readability
  • Scalable Text: Support for browser zoom up to 200% without horizontal scrolling
  • Alternative Text: Descriptive alt text for all images and graphics
  • Focus Indicators: Clear visual focus indicators for interactive elements
  • Semantic HTML: Proper heading structure and landmark regions

Mobile App Accessibility

Our applications support:

  • VoiceOver (iOS): Full compatibility with Apple's screen reader
  • TalkBack (Android): Comprehensive support for Android's accessibility service
  • Voice Control: Voice navigation and command support
  • Switch Control: External switch support for users with mobility impairments
  • Large Text: Dynamic font scaling up to accessibility sizes
  • High Contrast: Enhanced visual contrast options
  • Reduced Motion: Respect for reduced motion preferences

Visual Accessibility

  • Color Contrast: Minimum 4.5:1 contrast ratio for normal text, 3:1 for large text
  • Color Independence: Information not conveyed through color alone
  • Font Choices: Legible, accessible font families
  • Scalable Interface: Responsive design that adapts to different screen sizes
  • Dark Mode: Reduced eye strain option available

Motor Accessibility

  • Large Touch Targets: Minimum 44x44 point touch targets on mobile
  • Gesture Alternatives: Alternative input methods for complex gestures
  • Timeout Extensions: Adjustable time limits for timed content
  • Error Prevention: Clear error messages and correction suggestions

Cognitive Accessibility

  • Clear Navigation: Consistent and predictable navigation patterns
  • Simple Language: Clear, concise content writing
  • Error Handling: Helpful error messages and recovery options
  • Progress Indicators: Clear feedback on processes and loading states

Testing and Validation

Automated Testing

We use automated accessibility testing tools including:

  • axe-core: Automated accessibility testing engine
  • WAVE: Web accessibility evaluation tool
  • Lighthouse: Google's accessibility audit tool
  • Platform Tools: Xcode Accessibility Inspector, Android Accessibility Scanner

Manual Testing

Our accessibility testing includes:

  • Keyboard-only navigation testing
  • Screen reader testing with multiple assistive technologies
  • Color contrast verification
  • User testing with people who have disabilities

Third-Party Audits

We conduct regular accessibility audits with qualified accessibility consultants to identify and address potential barriers.

Known Limitations

While we strive for full accessibility, some limitations may currently exist:

  • Third-party content: Some embedded content from external providers may not meet our accessibility standards
  • Legacy features: Older features may be updated gradually to meet current standards
  • Complex interactions: Some advanced features may require additional accessibility enhancements

We are actively working to address these limitations in future updates.
